# Установка «Claude → Figma» с нуля

Пошаговая инструкция для **чистого компьютера** (Windows), где ничего не установлено.
Плагин переносит дизайн из Claude в Figma как редактируемые слои (фреймы, текст,
шейпы, вектора-иконки) — аналог html.to.design.

Время на всё: ~15 минут.

---

## Что понадобится (ставим по порядку)

| # | Программа | Зачем | Где взять |
|---|-----------|-------|-----------|
| 1 | **Figma Desktop** | где работает плагин | https://www.figma.com/downloads/ |
| 2 | **Node.js LTS** | движок MCP-сервера | https://nodejs.org (кнопка LTS) |
| 3 | **Claude Desktop** | чтобы просить «отправь в Figma» | https://claude.ai/download |
| 4 | **Папка проекта** | сам плагин + сервер | скопировать папку с этим файлом |

> Если Claude Desktop не нужен — можно работать **вручную** (вставлять HTML в окно
> плагина). Тогда пункт 3 пропусти и смотри «Вариант B».

---

## Шаг 1. Установить Figma Desktop
Скачай и установи с https://www.figma.com/downloads/, войди в аккаунт.

## Шаг 2. Установить Node.js
1. Зайди на https://nodejs.org, скачай **LTS**, установи (все галочки по умолчанию).
2. Проверь: открой **новое** окно PowerShell и выполни:
   ```powershell
   node --version
   ```
   Должна появиться версия (например `v22.x`). Если «не распознано» — закрой и
   открой терминал заново (PATH обновляется только в новых окнах).

## Шаг 3. Скопировать папку проекта
Положи папку проекта куда удобно, например в `Документы`. Внутри должны быть:
```
plugin/      ← сам плагин Figma
server/      ← MCP-сервер
SETUP.md     ← этот файл
README.md
```

## Шаг 4. Установить зависимости сервера
В PowerShell перейди в папку `server` и установи пакеты:
```powershell
cd "ПУТЬ\К\ПРОЕКТУ\server"
npm install
```
> Если npm ругается «выполнение сценариев отключено» — запусти вместо этого
> `npm.cmd install`, либо один раз разреши скрипты:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` (ответь `Y`).

## Шаг 5. Импортировать плагин в Figma
1. Открой Figma Desktop, создай/открой любой файл.
2. Меню **Plugins → Development → Import plugin from manifest…**
3. Выбери файл `plugin/manifest.json` из папки проекта.
4. Плагин появится в **Plugins → Development → Claude → Figma**.

Готово — дальше выбери один из трёх способов работы.

---

## Вариант A — через Claude Desktop (рекомендуется)

Самый удобный: просто просишь Claude — он сам отправляет дизайн в Figma.

1. Найди файл конфига Claude Desktop:
   `C:\Users\ИМЯ\AppData\Roaming\Claude\claude_desktop_config.json`
   (можно вставить в адресную строку проводника: `%APPDATA%\Claude`)
   Если файла нет — создай пустой с `{}` внутри.

2. Впиши в него блок `mcpServers` (если файл не пустой — добавь ключ `mcpServers`
   рядом с существующими, не удаляя их):
   ```json
   {
     "mcpServers": {
       "claude-to-figma": {
         "command": "node",
         "args": ["C:\\ПУТЬ\\К\\ПРОЕКТУ\\server\\mcp-server.js"]
       }
     }
   }
   ```
   ⚠️ Путь — **абсолютный**, со **сдвоенными** обратными слэшами `\\`.
   Если `node` не находится — укажи полный путь:
   `"command": "C:\\Program Files\\nodejs\\node.exe"`.

3. **Полностью перезапусти Claude Desktop** (выйти из трея у часов → открыть заново).
   Сервер он запустит сам.

4. Открой плагин в Figma (точка вверху позеленеет — значит мост подключён).

5. Проси в Claude:
   - *«проверь figma_status»* → должен ответить «плагин подключён»;
   - *«сгенерируй экран приложения и отправь в Figma»* или
     *«импортируй этот HTML в Figma»* (приложив разметку).

---

## Вариант B — вручную (без Claude, для проверки)

1. В PowerShell запусти сервер-мост (в **новом** окне, оставь открытым):
   ```powershell
   cd "ПУТЬ\К\ПРОЕКТУ\server"
   node mcp-server.js
   ```
   Появится `MCP (stdio) запущен. WebSocket-мост: ws://localhost:3055`.
2. Открой плагин в Figma — точка позеленеет.
3. Вставь HTML в текстовое поле плагина и нажми **«Импортировать в Figma»**
   (для пробы можно открыть `example.html` и скопировать его содержимое).

> Не запускай Вариант A и B одновременно — оба занимают порт 3055.

---

## Вариант C — claude.ai в браузере (продвинутый, через туннель)

Нужен план Claude **Pro/Max/Team** (на бесплатном кастомных коннекторов нет).
Подробно описано в `README.md`, раздел «claude.ai в браузере». Кратко:
1. Запусти HTTP-сервер с секретом:
   `$env:AUTH_TOKEN="длинный-секрет"; node mcp-http-server.js`
2. Подними туннель: `cloudflared tunnel --protocol http2 --url http://127.0.0.1:3056`
   (флаг `--protocol http2` важен, если провайдер режет UDP).
3. В claude.ai → Settings → Connectors → Add custom connector → вставь
   `https://<адрес-туннеля>/<секрет>/mcp`.
4. Открой плагин в Figma и проси импорт прямо в браузере.

## Шрифты (для точного совпадения)

Плагин запрашивает шрифты дизайна; если их нет в системе — Figma подменяет на
**Inter** (плагин напишет в уведомлении, какие именно подменил). Чтобы вёрстка
совпадала точь-в-точь, поставь нужные шрифты в Windows и перезапусти Figma.
Бесплатные обычно есть на Google Fonts (например Unbounded, Golos Text,
JetBrains Mono). Платные (например PP Gosha Sans) — только если они у тебя есть.

---

## Если что-то не так

| Симптом | Причина / решение |
|---|---|
| Точка в плагине красная | Сервер не запущен. Вариант A: перезапусти Claude Desktop. Вариант B: запусти `node mcp-server.js`. |
| `node не распознан` | Открой **новое** окно терминала после установки Node. |
| `выполнение сценариев отключено` | Используй `node ...` напрямую или `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`. |
| `EADDRINUSE :3055` | Порт занят: запущены сразу два сервера. Оставь один. |
| Claude не видит инструменты | Проверь путь в `claude_desktop_config.json` (абсолютный, `\\`) и перезапусти Claude. |
| Текст приходит шрифтом Inter | Нужного шрифта нет в системе — поставь его (см. раздел «Шрифты»). |
| Динамические подписи пустые | Это шаблоны `{{ }}` из артефакта Claude — без рантайма значений нет (это нормально). |

## Структура проекта (справочно)

```
plugin/
  manifest.json   манифест плагина Figma
  code.js         песочница Figma: строит слои (фреймы/текст/шейпы/вектора)
  ui.html         iframe: HTML → дерево узлов (рендер + замер стилей)
server/
  mcp-server.js   MCP по stdio (Claude Desktop / Claude Code)
  mcp-http-server.js  MCP по HTTP (claude.ai через туннель)
  bridge.js       WebSocket-мост к плагину (порт 3055)
  tools.js        инструменты import_html / import_url / figma_status
  package.json
example.html      демо-разметка для ручной проверки
```
