# Облачная версия (как html.to.design)

Цель: плагин ставится из Figma Community, и «Claude → Figma» работает у любого
пользователя **без локального сервера**. Вместо `ws://localhost:3055` — один
облачный relay, а пользователи разводятся по **комнатам** (pairing-код).

## Принцип

```
claude.ai / Claude Desktop                Figma (у пользователя)
        │                                        │
        │ MCP HTTPS                               │ WSS
        ▼                                         ▼
   https://relay.домен/<КОД>/mcp  ───room<КОД>───  wss://relay.домен/plugin?room=<КОД>
                         \____________ RELAY (облако) ____________/
                          маршрутизация задания в нужную комнату
```

- **Плагин** при открытии генерирует КОД (или пользователь логинится), подключается
  к relay в комнату `<КОД>` и показывает готовый URL коннектора.
- **Claude** добавляет этот URL как Connector. Его вызовы `import_html` уходят на
  relay, тот находит комнату `<КОД>` и шлёт задание в открытый плагин этого юзера.
- Плагин строит слои локально в Figma (как сейчас) и возвращает результат.

## Компоненты

### 1. Relay-сервис (Node, в облаке)
Почти целиком переиспользует текущий код (`bridge.js` + `tools.js` +
`mcp-http-server.js`), но **ключует всё по комнате**, а не по одному глобальному
соединению:
- `wss://relay/plugin?room=КОД` — сюда коннектится плагин; сокет кладётся в `rooms[КОД]`.
- `https://relay/КОД/mcp` — StreamableHTTP MCP для Claude; `sendToPlugin` шлёт
  задание только сокетам из `rooms[КОД]` и ждёт ответ по `reqId`.
- В памяти: `Map<код, Set<socket>>` + `Map<reqId, resolve>`. Это ~+40 строк к текущему.

### 2. Плагин (правки в `ui.html`)
- WS-адрес: вместо `ws://localhost:3055` → `wss://relay.домен/plugin?room=<КОД>`
  (адрес relay зашит, КОД генерируется/хранится в `figma.clientStorage`).
- UI: показать **код пары** и кнопку «Скопировать URL коннектора»
  (`https://relay.домен/<КОД>/mcp`).
- Ручная вставка HTML работает как раньше (relay для неё не нужен).

### 3. Pairing (связка Claude ↔ плагин)
- **MVP:** КОД = случайный секрет в URL (как сейчас токен, но relay облачный).
  Плагин генерит, юзер вставляет URL в Connectors. Просто и без аккаунтов.
- **Продукт:** аккаунты/логин + персональные API-ключи, КОД привязан к юзеру,
  ротация, срок жизни.

## Хостинг
Нужен always-on хост с **WebSocket + TLS** (wss/https):
- **Fly.io** / **Railway** / **Render** — есть дешёвые/бесплатные тиры, WS из коробки, TLS авто.
- или **VPS** (Hetzner/…): Node + Caddy/nginx для TLS.
Домен: можно поддомен хостинга, можно свой. Один инстанс тянет много комнат.

## Безопасность
- КОД — длинный неугадываемый секрет; кто знает URL, тот шлёт в твою Figma → как пароль.
- Лимиты: размер payload (HTML), rate-limit на комнату, таймауты.
- claude.ai-коннекторы могут требовать **OAuth** — для продукта прикрутить OAuth/аккаунты;
  для MVP — неаутентифицированный remote MCP с секретом в пути (как уже проверено).
- Логи без секретов (токен/код не писать в stdout).

## Что переиспользуется
| Сейчас | В облаке |
|---|---|
| `bridge.js` (один Set плагинов) | rooms: `Map<код, Set>` |
| `tools.js` (import_html/url/status) | без изменений, `sendToPlugin` принимает `room` |
| `mcp-http-server.js` (StreamableHTTP) | то же, эндпоинт `/<код>/mcp` |
| `ui.html` ws→localhost | ws→`wss://relay/plugin?room=КОД` + UI кода |

## Деплой на Fly.io (Фаза 1 — готово к запуску)

Relay уже собран: `server/cloud-relay.js` (+ `Dockerfile`, `fly.toml`, `.dockerignore`).
Проверен локально сквозным тестом (MCP + маршрутизация по комнатам + изоляция).

```powershell
# 1. установить flyctl
powershell -c "iwr https://fly.io/install.ps1 -useb | iex"
# 2. вход/регистрация
fly auth signup        # или: fly auth login
# 3. из папки server (имя app в fly.toml глобально уникально — при конфликте поменяй)
cd "C:\...\07 plugin figma\server"
fly launch --no-deploy --copy-config --name claude-to-figma-relay
fly deploy
```
После деплоя получишь хост вида `https://claude-to-figma-relay.fly.dev`.

**Подключение (Фаза 1, без UI-кода):**
- В плагине поле WebSocket → `wss://<хост>/plugin?room=<КОД>` (любой свой секретный КОД).
- В claude.ai Connectors → `https://<хост>/<КОД>/mcp`.
- КОД в обоих местах одинаковый — это и есть «комната».

**Фаза 2 (UX):** перенести генерацию/хранение КОДА и кнопку «копировать URL коннектора»
в сам плагин (`ui.html` + `figma.clientStorage`), чтобы не вводить адрес руками.
